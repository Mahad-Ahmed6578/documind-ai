End-to-end pipeline with 3 swappable embedding providers (TF-IDF / GLM-4 Semantic / Gemini), 2 vector stores (in-memory / Pinecone), document-scoped queries, comparison boost, and localStorage message persistence

2Ingestion Pipeline (Background, < 3s for 10-page PDF)
Parse documentdocument-parser.ts — pdf-parse / mammoth / UTF-8

Embed chunks (3 options)embeddings.ts dispatches to TF-IDF (256d), GLM-4 Semantic (512d), or Gemini (768d)
Persist to Prisma + Vector DB (2 options)Prisma (always) + in-memory or Pinecone (if VECTOR_STORE_PROVIDER=pinecone); status → "ready"

🔀 Full Swappability — 3 Embedders × 2 Vector Stores × 2 LLMs

Edit .env to switch any layer without code changes:

# LLM (chat responses)
LLM_PROVIDER=zai    # or "gemini"

# Embeddings (text → vectors)
EMBED_PROVIDER=glm4    # "tfidf" (256d), "glm4" (512d), or "gemini" (768d)

# Vector Store (where to store/search)
VECTOR_STORE_PROVIDER=memory    # or "pinecone"

12 possible combinations — all work without code changes. Defaults: Z.ai GLM-4 + GLM-4 Semantic + in-memory (zero API keys needed).
